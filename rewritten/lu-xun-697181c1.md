人有一种本领。能把任何苦差事说成是福气。

从前是手工抄写，后来是 Excel 表格，现在是 Agent。工具换了三代，话术没变：「效率」「解放」「专注真正重要的事」。

不过这一回，似乎真有点不同。

---

有人用三个 Agent 做高考志愿填报。数据分析师挖数据，择校顾问出方案，报告撰写员写报告。三条命令，一条链，自动流转。

听起来像科幻。做起来像流水线。

数据分析师去院校官网、阳光高考平台抓录取分数。择校顾问综合学科评估、就业数据、城市偏好，排出冲稳保七所学校。报告撰写员把方案翻译成人话，装进一份报告里。

人干什么呢？人定需求，写任务描述，审结果，做决策。

「Agent 是手。你是脑。」

这话听着体面。仿佛人终于从劳役中解脱，可以去做「真正重要的事」了。

---

然而我疑心。

所谓「真正重要的事」，究竟是什么？

定需求——把「帮我分析志愿」换成「理科 635 分，省排名 3200，意向计算机，想留长三角」。这叫定需求。说穿了，就是把模糊的念头翻译成机器听得懂的话。

写任务描述——告诉 Agent 数据来源优先级：院校官网优于阳光高考平台优于省考试院。每条数据标注 URL 和年份。输出保存到指定路径。

审结果——看 Agent 交上来的东西，挑毛病，让它改。

做决策——在七所学校里选一所。

这四件事，哪一件不需要动脑子？哪一件可以偷懒？

从前一个人查数据、做分析、写报告，累是累，好歹全程清醒。现在三个 Agent 分头干，人坐在中间「审结果、做决策」，看似轻松，实则更难。

因为你必须比三个 Agent 都懂。否则你怎么审？你怎么知道数据分析师漏了哪所院校？你怎么知道择校顾问的建议是深思熟虑还是敷衍了事？

工具越强，对用工具的人要求越高。

这不是解放。这是筛选。

---

文章里还列了一串「容易踩的坑」。

Profile 工具给太多，Agent 就会分心。任务描述没写输出路径，下一个 Agent 就找不到输入。依赖关系搞反了，整个链条卡死。Gateway 没开，就没人派活。

每一条都是血泪。

然而这些血泪，恰恰说明一件事：Agent 不是魔法。它是机器。机器需要人喂料，需要人调教，需要人擦屁股。

那些以为配三条命令就能躺平的人，迟早会在某个深夜盯着「task-001 状态：error」的界面，想起这篇文章，然后苦笑。

---

不过话说回来，这文章确实指出了一个真问题。

人的时间花在哪里？

从前花在执行上。查数据、做表格、写报告。重复劳动，日复一日。

现在执行可以交给 Agent。人的时间应该花在定需求、做决策、担责任上。

这三件事，恰恰是大多数人最不愿意做的。

定需求意味着想清楚自己要什么。做决策意味着承担后果。担责任意味着出了问题不能甩锅给工具。

人喜欢执行，因为执行不用承担后果。表格做错了，改就是了。报告写得不好，重写就是了。

但决策错了，就是错了。没有「重来」按钮。

Agent 把人从执行中解放出来，不是为了让人闲着。是为了让人面对真正的难题。

这才是这篇文章没说出口的话。

---

最后讲一个细节。

文章说 Agent 的数据来自公开互联网搜索，不是内置数据库。需要登录才能看的付费平台数据，它拿不到。

这意味着什么？

意味着 Agent 能做的，是一个会上网搜索的普通人能做的事。它不会比你聪明，不会比你知道得多。它只是比你快，比你勤快，比你不会累。

快、勤快、不会累。这三样东西加在一起，确实可怕。

但可怕之处不在于 Agent。可怕之处在于：如果你的工作只需要快、勤快、不会累就能完成，那你的工作本来就不该由人来做。

Agent 不是在抢你的饭碗。Agent 是在告诉你，你的饭碗里装的是什么。

以上。
